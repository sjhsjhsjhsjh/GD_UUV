# ACNet (Acoustic Concealment Network) 网络设计说明

## 一、网络概述

**ACNet** 是一个用于 UUV（无人水下航行器）隐蔽突防任务的决策网络，采用 PyTorch 实现的 Actor-Critic 架构。该网络能够同时处理**3D 空间信息**（声场传播损失与地形可通性）和**6 维状态向量**（UUV 与敌方坐标），输出**6 个离散动作**的策略分布和**状态价值估计**。

### 核心特征

- **双输入融合架构**：空间分支处理地形与声学特征，向量分支处理位置状态信息
- **GPU 强制推理**：所有计算必须在 CUDA GPU 上执行，不支持 CPU 降级
- **Actor-Critic 双头输出**：同时生成动作策略与价值评估
- **实时性设计**：卷积与线性层权衡，支持批量推理加速

## 二、网络架构

### 2.1 整体结构

```
┌─────────────────────────────────────────────────────────────┐
│                     ACNet 整体架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  Spatial Input (B, 2, D, H, W)    State Vector (B, 6)       │
│         │                                 │                   │
│         ├─────────────────┬───────────────┘                  │
│         │                 │                                   │
│    ┌────▼────┐      ┌────▼────┐                              │
│    │  Spatial │      │  Vector  │                              │
│    │ Branch   │      │  Branch  │                              │
│    │ (Conv3D) │      │  (MLP)   │                              │
│    └────┬─────┘      └────┬─────┘                              │
│         │                 │                                   │
│    (B, 128)         (B, 64)                                   │
│         │                 │                                   │
│         └────────┬────────┘                                   │
│                  │                                           │
│            ┌─────▼──────┐                                    │
│            │ Cat + FC   │  (B, 192) → (B, 256)              │
│            │ + LayerNorm│                                    │
│            └─────┬──────┘                                    │
│                  │                                           │
│         ┌────────┴──────────┐                               │
│         │                   │                               │
│    ┌────▼─────┐      ┌─────▼────┐                           │
│    │ Actor    │      │ Critic   │                           │
│    │ Head     │      │ Head     │                           │
│    │ (B, 6)  │      │ (B, 1)  │                           │
│    └──────────┘      └──────────┘                           │
│                                                               │
│  Actor Logits (B, 6)         State Value (B, 1)             │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 空间分支（Spatial Branch）

**功能**：从 3D 体素输入提取空间特征，处理声学与地形信息。

**结构**：
| 层级 | 输入 | 操作 | 输出 | 备注 |
|------|------|------|------|------|
| 1 | (B, 2, D, H, W) | Conv3d(k=5) + BN + ReLU | (B, 32, D, H, W) | 核心尺寸 5，保留空间细节 |
| 2 | (B, 32, D, H, W) | Conv3d(k=3) + BN + ReLU | (B, 64, D, H, W) | 细粒度特征提取 |
| 3 | (B, 64, D, H, W) | Conv3d(k=3) + BN + ReLU | (B, 128, D, H, W) | 高阶特征压缩 |
| GAP | (B, 128, D, H, W) | AdaptiveAvgPool3d | (B, 128, 1, 1, 1) | 全局平均池化 |
| Flatten | (B, 128, 1, 1, 1) | flatten(start_dim=1) | (B, 128) | 特征向量化 |

**重要性质**：
- 核心尺寸从 5 逐步降低到 3，避免过度下采样
- Padding 策略保持空间维度，输出尺寸始终为 (D, H, W)
- 全局平均池化对输入分辨率不敏感，支持变长输入
- 通道数递增（2 → 32 → 64 → 128），逐步抽象特征

**输入通道含义**：
- 通道 0：传播损失 (TL, Transmission Loss) 图 — 声波在水中的衰减规律
- 通道 1：地形可通性 (Passability) — 二值或概率地图，指示地形障碍

### 2.3 向量分支（Vector Branch）

**功能**：编码 6 维状态向量（坐标与敌方位置），提取相对位置特征。

**结构**：
```
(B, 6) ──[Linear(64)]──[ReLU]──[Linear(64)]──[ReLU]──→ (B, 64)
        ↑                                              ↑
    输入：(x, y, z, x_e, y_e, z_e)      输出：位置编码特征
```

**设计理由**：
- 两层隐藏层足以捕捉坐标间的相对关系（距离、方向）
- 64 维宽度与空间分支 128 维特征量级匹配
- ReLU 激活提供非线性表达能力

### 2.4 融合层（Fusion Layer）

**功能**：合并两个分支的特征，增强表达与稳定性。

**结构**：
```
Spatial Feature (B, 128)  Vector Feature (B, 64)
        │                           │
        └──────────┬────────────────┘
                   │
            ┌──────▼──────┐
            │ Concatenate │  (B, 192)
            └──────┬──────┘
                   │
            ┌──────▼──────────┐
            │ Linear(192→256) │
            └──────┬──────────┘
                   │
            ┌──────▼──────────┐
            │ LayerNorm(256)  │  → 稳定特征分布
            └──────┬──────────┘
                   │
            ┌──────▼──────────┐
            │ ReLU            │
            └──────┬──────────┘
                   │
             (B, 256) 融合特征
```

**设计细节**：
- **Concatenate**：简单有效的特征融合方式，保留两分支的独立信息
- **Linear(192→256)**：升维融合，256 维足以承载两分支信息并为双头分化
- **LayerNorm**：规范化融合特征的均值与方差，提升训练稳定性和收敛速度
- **ReLU**：引入非线性，增强表达能力

### 2.5 Actor-Critic 双输出头

#### Actor 头（动作策略头）
```
Fused Feature (B, 256)
        │
    ┌───▼────────┐
    │ Linear(6)  │  ← 6 个离散动作
    └───┬────────┘
        │
  Actor Logits (B, 6)
  ├─ 索引 0: 前进 (+X)
  ├─ 索引 1: 后退 (-X)
  ├─ 索引 2: 左转 (+Y)
  ├─ 索引 3: 右转 (-Y)
  ├─ 索引 4: 上升 (+Z)
  └─ 索引 5: 下降 (-Z)
```

**输出**：未归一化的 logits，维度 (B, 6)  
**后续处理**：在策略模块中使用 `Softmax` 转换为概率分布，或使用 `Gumbel-Max` 采样

#### Critic 头（价值估计头）
```
Fused Feature (B, 256)
        │
    ┌───▼────────┐
    │ Linear(1)  │
    └───┬────────┘
        │
  State Value (B, 1)  → V(s) 状态价值估计
```

**输出**：单一标量价值，维度 (B, 1)  
**解释**：当前状态的长期折扣累计奖励期望值，用于价值函数梯度估计或 TD 误差计算

## 三、输入输出接口

### 3.1 输入定义

#### 空间输入（Spatial Input）
```python
spatial_input: torch.Tensor
  形状: (Batch, 2, Depth, Height, Width)
  dtype: torch.float32
  device: GPU (CUDA)
  
  取值范围示例：
  ├─ TL 图 (通道 0)       : [0, 100] (单位：dB)
  └─ Passability (通道 1) : [0, 1] 或 {0, 1} (概率或二值)
```

**常见分辨率**：
- 小规模测试：(16, 16, 16)
- 中等规模任务：(20, 24, 28)
- 大规模精密操作：(32, 40, 50)

#### 状态向量（State Vector）
```python
state_vector: torch.Tensor
  形状: (Batch, 6)
  dtype: torch.float32
  device: GPU (CUDA)
  
  内容：[x, y, z, x_e, y_e, z_e]
  ├─ x, y, z      : UUV 当前坐标（水下相对坐标系）
  └─ x_e, y_e, z_e: 敌方 / 目标当前坐标
  
  建议归一化范围：[-1, 1] 或 [0, 1]（相对于环境尺寸）
```

### 3.2 输出定义

#### Actor Logits（动作策略）
```python
actor_logits: torch.Tensor
  形状: (Batch, 6)
  dtype: torch.float32
  device: GPU (CUDA)
  
  语义：未归一化动作 logits
  后续处理：
    ├─ 概率分布：probs = torch.softmax(actor_logits, dim=-1)
    ├─ 对数概率：log_probs = torch.log_softmax(actor_logits, dim=-1)
    ├─ 动作采样：action = torch.multinomial(probs, num_samples=1)
    └─ 贪心选择：action = torch.argmax(actor_logits, dim=-1)
```

#### Critic Value（状态价值）
```python
state_value: torch.Tensor
  形状: (Batch, 1)
  dtype: torch.float32
  device: GPU (CUDA)
  
  语义：V(s) 状态价值估计（折扣累计奖励期望）
  
  应用场景：
    ├─ TD 误差：delta = reward + gamma * V(s') - V(s)
    ├─ 优势函数：A = Q - V(s)
    ├─ GAE：λ-return 计算
    └─ 价值函数损失：loss_critic = (target_value - V(s))^2
```

### 3.3 Batch 处理

ACNet 支持灵活的 Batch 大小：

```python
# 单样本推理（Batch=1）
spatial_input = torch.randn(1, 2, 20, 24, 28, device="cuda")
state_vector = torch.randn(1, 6, device="cuda")

# → actor_logits.shape = (1, 6), state_value.shape = (1, 1)

# 批量推理（Batch=32，典型训练批次）

state_vector = torch.randn(32, 6, device="cuda")
actor_logits, state_value = model(spatial_input, state_vector)
# → actor_logits.shape = (32, 6), state_value.shape = (32, 1)

# 大规模批量推理（Batch=256，用于高吞吐）

state_vector = torch.randn(256, 6, device="cuda")
actor_logits, state_value = model(spatial_input, state_vector)
# → actor_logits.shape = (256, 6), state_value.shape = (256, 1)
```

| 组件 | 参数数量 | 说明 |
|------|---------|------|
| 空间分支 Conv3d | ~143K | 3 层卷积 + BN |

| 融合层 | ~49K | FC + LN |

### 4.2 计算复杂度

**推理时间估算**（单样本，GPU: NVIDIA A100）：
- 空间分支前向：~2-3 ms
- 向量分支前向：<0.1 ms
- 融合层 + 双头：<0.5 ms
- **总计**：~2-3 ms/sample

**内存占用**（推理态）：
- 模型权重：~1 MB
- Batch=32 激活：~50 MB
- 总计：~51 MB（GPU 显存占用极低）

## 五、使用示例

### 5.1 基础使用

```python
import torch
from agent import ACNet

# 初始化模型并移到 GPU
device = "cuda"
model = ACNet(device=device).to(device)

# 构造输入
batch_size = 4
spatial_input = torch.randn(batch_size, 2, 20, 24, 28, device=device)
state_vector = torch.randn(batch_size, 6, device=device)

# 前向传播
actor_logits, state_value = model(spatial_input, state_vector)

# 获取动作与价值
action_probs = torch.softmax(actor_logits, dim=-1)
action = torch.multinomial(action_probs, num_samples=1)  # (B, 1)
value = state_value.squeeze(-1)  # (B,)

print(f"Action: {action.shape}, Value: {value.shape}")
```

## 六、重要约束与注意事项

### 6.1 GPU 强制执行

**约束**：所有推理必须在 GPU 上执行，不支持 CPU 降级。

**违规示例**（会抛出异常）：
```python
model = ACNet(device="cuda")

# ❌ 错误：输入在 CPU 上
spatial_input = torch.randn(2, 2, 20, 24, 28)  # device='cpu'
state_vector = torch.randn(2, 6)

try:
    actor_logits, state_value = model(spatial_input, state_vector)
except RuntimeError as e:
    print(e)  # "spatial_input 必须在 GPU 上执行..."
```

**正确做法**：
```python
# ✓ 正确：显式指定 device='cuda'
spatial_input = torch.randn(2, 2, 20, 24, 28, device="cuda")
state_vector = torch.randn(2, 6, device="cuda")
actor_logits, state_value = model(spatial_input, state_vector)
```

### 6.2 设备初始化

```python
# ✓ 推荐：创建时指定设备
model = ACNet(device="cuda").to("cuda")

# ✗ 错误：尝试 CPU 推理
model = ACNet(device="cpu")  # 抛出 ValueError

# ✓ 多卡场景
model = ACNet(device="cuda:0").to("cuda:0")
# 或
model = ACNet(device="cuda:1").to("cuda:1")
```

### 6.3 数据类型与范围

| 参数 | 建议值 | 说明 |
|------|--------|------|
| dtype | float32 | PyTorch 默认，与网络权重一致 |
| 空间输入范围 | [0, 100] (TL) | 模型对绝对值不敏感，归一化可加速收敛 |
| 状态向量范围 | [-1, 1] | 相对坐标，中心化为零有利于网络学习 |
| Batch 大小 | 1-256 | 灵活支持，建议 >= 32 用于稳定训练 |

### 6.4 梯度与反向传播

ACNet 完全支持自动微分（autograd）：

```python
model = ACNet(device="cuda").to("cuda")
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

spatial_input = torch.randn(8, 2, 20, 24, 28, device="cuda", requires_grad=False)
state_vector = torch.randn(8, 6, device="cuda", requires_grad=False)

# 前向
actor_logits, state_value = model(spatial_input, state_vector)

# 定义损失
actor_loss = -torch.log_softmax(actor_logits, dim=-1)[range(8), actions].mean()
critic_loss = torch.nn.functional.mse_loss(state_value.squeeze(-1), targets)
total_loss = actor_loss + 0.5 * critic_loss

# 反向与优化
optimizer.zero_grad()
total_loss.backward()
optimizer.step()
```

## 七、性能优化建议

### 7.1 推理优化

1. **批量推理**：使用尽可能大的 batch 大小（受 GPU 内存限制）
2. **混合精度**：在不影响精度的场景下，考虑 FP16
3. **模型量化**：部署时可尝试 INT8 量化（需离线校准）

### 7.2 训练优化

1. **学习率调度**：使用 `ReduceLROnPlateau` 或 `CosineAnnealingLR`
2. **梯度剪裁**：防止爆炸梯度
   ```python
   torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
   ```
3. **权重衰减**：L2 正则化，在 optimizer 中设置 `weight_decay=1e-4`

### 7.3 稳定性增强

1. **特征归一化**：空间输入、状态向量预处理标准化
2. **梯度归一化**：在优化器中设置 `torch.nn.utils.clip_grad_norm_`
3. **熵正则化**（仅 Actor）：鼓励策略探索
   ```python
   entropy = -(action_probs * log_probs).sum(dim=-1).mean()
   actor_loss = -(log_probs * advantages).mean() - 0.01 * entropy
   ```

## 八、故障排查与常见错误

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `spatial_input 必须在 GPU 上执行` | 输入在 CPU | `spatial_input.to("cuda")` |
| `state_vector 必须为形状 (B, 6)` | 状态向量维度错误 | 检查 `state_vector.shape` |
| `spatial_input 通道数必须为 2` | 通道数不是 2 | 确保输入为 `(B, 2, D, H, W)` |
| `RuntimeError: CUDA out of memory` | 显存不足 | 减小 batch 大小或输入分辨率 |
| `CUDA 不可用` | 环境未安装 GPU 版本 PyTorch | 验证 `torch.cuda.is_available()` |
